# Linux چیست؟

Linux یک سیستم‌عامل متن‌باز (Open Source) است که بر پایه Linux Kernel ساخته شده است.

برخلاف تصور رایج، Linux فقط Kernel است.

آنچه ما روزانه از آن استفاده می‌کنیم معمولاً ترکیبی از موارد زیر است:

- Linux Kernel
    
- GNU Tools
    
- Shell
    
- Package Manager
    
- System Libraries
    
- User Space Programs
    

به همین دلیل بسیاری از متخصصان از عبارت **GNU/Linux** استفاده می‌کنند.

---

# سیستم‌عامل چیست؟

Operating System (OS)

نرم‌افزاری است که بین سخت‌افزار و برنامه‌های کاربردی قرار می‌گیرد.

بدون سیستم‌عامل، برنامه‌ها قادر به استفاده از:

- CPU
    
- RAM
    
- Disk
    
- Network
    
- USB
    
- Keyboard
    
- Display
    

نخواهند بود.

---

# Kernel چیست؟

Kernel قلب سیستم‌عامل است.

تمام ارتباط نرم‌افزارها با سخت‌افزار از طریق Kernel انجام می‌شود.

وظایف Kernel:

- مدیریت CPU
    
- مدیریت RAM
    
- مدیریت Processها
    
- مدیریت Storage
    
- مدیریت Network
    
- مدیریت Device Driverها
    
- مدیریت امنیت
    

تقریباً هر عملی که در Linux انجام می‌دهیم، در نهایت به Kernel ختم می‌شود.

---

# User Space چیست؟

برنامه‌هایی که کاربر اجرا می‌کند در User Space قرار دارند.

نمونه‌ها:

- Bash
    
- Nano
    
- Vim
    
- Docker
    
- Nginx
    
- Firefox
    

این برنامه‌ها مستقیماً با سخت‌افزار صحبت نمی‌کنند.

تمام درخواست‌های آنها ابتدا وارد Kernel می‌شود.

# Distribution چیست؟

Kernel به تنهایی قابل استفاده نیست.

شرکت‌ها و جامعه متن‌باز ابزارهای مختلفی را کنار Kernel قرار می‌دهند و یک Distribution می‌سازند.

نمونه‌ها:

- Ubuntu
    
- Debian
    
- Fedora
    
- Rocky Linux
    
- AlmaLinux
    
- Arch Linux
    
- openSUSE
    

---

# چرا Ubuntu؟

در این دوره از Ubuntu Server استفاده می‌کنیم زیرا:

- مستندات بسیار زیاد
    
- جامعه کاربری بزرگ
    
- مناسب یادگیری
    
- پشتیبانی طولانی مدت (LTS)
    
- سازگاری بسیار خوب با Docker
    
- سازگاری عالی با Proxmox
    
- سازگاری با Mailcow
    

---

# LTS چیست؟

Long Term Support

نسخه‌هایی که چندین سال Update امنیتی دریافت می‌کنند.

برای سرورها تقریباً همیشه استفاده از نسخه‌های LTS پیشنهاد می‌شود.

---

# Linux Server و Linux Desktop

Linux Desktop

برای کاربران عادی طراحی شده است.

دارای GUI است.

---

Linux Server

برای مدیریت سرورها طراحی شده است.

معمولاً GUI ندارد.

تمام مدیریت آن توسط Terminal انجام می‌شود.

---

# CLI چیست؟

Command Line Interface

رابط متنی سیستم‌عامل.

تقریباً تمام مدیرهای سیستم از CLI استفاده می‌کنند.

مزایا:

- سریع‌تر
    
- مصرف RAM کمتر
    
- قابل Script نویسی
    
- مناسب مدیریت از راه دور
    
- قابل خودکارسازی
    

---

# GUI چیست؟

Graphical User Interface

رابط گرافیکی.

برای Desktop مناسب است.

ولی اکثر سرورهای Linux بدون GUI نصب می‌شوند.

---

# Shell چیست؟

Shell واسط بین کاربر و Kernel است.

کاربر فرمان را وارد می‌کند.

↓

Shell آن را تفسیر می‌کند.

↓

↓

# Bash چیست؟

Bash

(Bourne Again Shell)

رایج‌ترین Shell در Linux است.

تقریباً تمام آموزش‌های این دوره بر پایه Bash خواهند بود.

---

# SSH چیست؟

Secure Shell

پروتکل مدیریت امن سرورها.

تقریباً تمام سرورهای Linux دنیا از طریق SSH مدیریت می‌شوند.

در آینده تقریباً تمام کارهای ما از طریق SSH خواهد بود.

---

# Open Source چیست؟

کد منبع برنامه در اختیار همه قرار دارد.

مزایا:

- شفافیت
    
- امنیت

- امنیت

# تفاوت Linux و Windows

Linux

- متن‌باز
    
- مدیریت Package متمرکز
    
- همه چیز فایل است.
    
- CLI محور
    
- مناسب سرورها

- اختصاصی (Proprietary)

- GUI محور
    
- مناسب Desktop

# Best Practices

- از همان ابتدا Terminal را دوست خود بدانید.
    
- مفاهیم را حفظ نکنید؛ دلیل وجود آنها را درک کنید.
    
- همیشه بدانید هر دستور با کدام بخش سیستم‌عامل کار می‌کند.
    
- قبل از حفظ کردن دستورها، معماری Linux را درک کنید.
    

---

# Common Mistakes

- تصور اینکه Linux فقط Ubuntu است.
    
- تصور اینکه Linux فقط یک سیستم‌عامل است، در حالی که Kernel بخش اصلی آن است.
    
- ترس از CLI.
    
- تلاش برای استفاده از Linux مانند Windows.
    
- حفظ کردن دستورها بدون درک مفهوم آنها.
