# فصل ۸ - کاربران و مجوزها | خلاصه

## مفاهیم کاربر
- **UID**: شناسه عددی کاربر (root=0، سیستم=1-999، عادی=1000+)
- **GID**: شناسه عددی گروه اصلی کاربر
- **passwd**: فایل اطلاعات کاربران در `/etc/passwd`
- **shadow**: رمزهای رمزنگاری‌شده در `/etc/shadow`

## ساختار /etc/passwd
```
username:x:UID:GID:comment:home:shell
root    :x:0  :0  :root   :/root:/bin/bash
```

## مجوزهای فایل
```
-rwxr-xr--  1  user  group  size  date  name
│└┬┘└┬┘└┬┘
│ │  │  └── others: r--  = 4
│ │  └───── group:  r-x  = 5
│ └──────── user:   rwx  = 7
└────────── نوع فایل (- فایل، d پوشه، l لینک)
```

## مقادیر عددی مجوز
| عدد | مجوز |
|-----|------|
| `7` | rwx - خواندن + نوشتن + اجرا |
| `6` | rw- - خواندن + نوشتن |
| `5` | r-x - خواندن + اجرا |
| `4` | r-- - فقط خواندن |
| `0` | --- - بدون مجوز |

## بیت‌های خاص
| بیت | روی فایل | روی پوشه |
|-----|---------|---------|
| **SUID** | با مجوز مالک اجرا می‌شود | بی‌تأثیر |
| **SGID** | با مجوز گروه اجرا می‌شود | فایل‌های جدید گروه پوشه را ارث می‌برند |
| **Sticky** | بی‌تأثیر | فقط مالک می‌تواند فایل خودش را حذف کند |

## umask
مجوزهایی که هنگام ساخت فایل جدید **حذف** می‌شوند:
- پیش‌فرض umask = 022
- فایل جدید: 666 - 022 = **644**
- پوشه جدید: 777 - 022 = **755**
