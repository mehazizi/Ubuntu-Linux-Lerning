# فصل ۲۴ - امنیت | خلاصه

## اصول امنیت لینوکس
- **کمترین دسترسی**: هر کاربر/سرویس فقط به آنچه نیاز دارد
- **دفاع لایه‌ای**: چندین لایه امنیتی مستقل
- **کمترین سطح حمله**: بستن همه چیزهای غیرضروری
- **ممیزی**: ثبت همه رویدادهای مهم

## PAM چیست؟
Pluggable Authentication Modules - چارچوب احراز هویت لینوکس که می‌توان آن را سفارشی کرد.

## سیاست رمز عبور قوی
```
حداقل ۱۲ کاراکتر
ترکیب حروف بزرگ، کوچک، عدد، نماد
عدم استفاده مجدد از رمزهای قبلی
تغییر دوره‌ای
```

## ممیزی با auditd
سیستم ثبت رویدادهای امنیتی هسته لینوکس برای:
- تغییر فایل‌های حساس
- دسترسی کاربران
- فراخوانی‌های سیستمی مشکوک
