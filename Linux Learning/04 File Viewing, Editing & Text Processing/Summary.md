# مشاهده فایل‌ها

برای مشاهده فایل‌ها ابزارهای مختلفی وجود دارد.

هرکدام کاربرد خاص خود را دارند.

مهم‌ترین آنها:

- cat
    
- less
    
- more
    
- head
    
- tail
    

---

# cat

کل فایل را یکجا نمایش می‌دهد.

مناسب فایل‌های کوچک.

برای فایل‌های بزرگ مناسب نیست.

---

# less

بهترین ابزار مشاهده فایل‌های بزرگ.

قابلیت‌ها:

- جستجو
    
- حرکت بین صفحات
    
- حرکت به ابتدا و انتها
    
- خروج بدون بارگذاری کامل فایل در حافظه
    

در عمل تقریباً همیشه less از more بهتر است.

---

# Redirection

خروجی دستورات را می‌توان داخل فایل ذخیره کرد.

به این عمل Redirection گفته می‌شود.

---

# >

اگر فایل وجود نداشته باشد:

ایجاد می‌شود.

اگر وجود داشته باشد:

کاملاً جایگزین می‌شود.

---

به انتهای فایل اضافه می‌کند.

اطلاعات قبلی حذف نمی‌شوند.

# Pipe

یکی از مهم‌ترین قابلیت‌های Linux.

Pipe خروجی یک دستور را مستقیماً وارد دستور بعدی می‌کند.

نماد آن:

```text
|
```

به جای ساخت فایل موقت، برنامه‌ها مستقیماً با یکدیگر ارتباط برقرار می‌کنند.

# Best Practices

- قبل از ویرایش فایل‌های مهم از آنها Backup بگیرید.
    
- برای مشاهده فایل‌های بزرگ از **less** استفاده کنید.
    
- برای مانیتورینگ Logها از **tail -f** استفاده کنید.
    
- هنگام تغییر فایل‌های داخل **/etc** دقت بسیار بالایی داشته باشید.
    
- ابتدا با Nano مسلط شوید، سپس Vim را به‌صورت حرفه‌ای یاد بگیرید.
    

---

# Common Mistakes

- استفاده از `cat` برای فایل‌های بسیار بزرگ.
    
- اشتباه گرفتن `>` و `>>`.
    
- ویرایش مستقیم فایل‌های سیستمی بدون Backup.
    
- خروج از Vim بدون ذخیره یا برعکس.
    
- نادیده گرفتن پیام‌های خطا (stderr).
    
- فراموش کردن استفاده از Pipe و ساخت فایل‌های موقت غیرضروری.
    

---

# Production Tips

- بیشتر عیب‌یابی‌های واقعی با ترکیب `tail`, `grep`, `less` و Pipe انجام می‌شود.
    
- یادگیری Vim یک سرمایه‌گذاری بلندمدت است؛ تقریباً روی هر سرور لینوکسی در دسترس است.
    
- توانایی خواندن سریع Logها یکی از مهم‌ترین مهارت‌های Linux Administrator است.
    
- سعی کنید به جای حفظ دستورات، منطق جریان داده (stdin → stdout → Pipe → فایل) را درک کنید؛ این دیدگاه در تمام ابزارهای لینوکس کاربرد خواهد داشت.
