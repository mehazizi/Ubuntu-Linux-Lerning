# فصل ۲۵ - کانتینر و Docker | خلاصه

## کانتینر چیست؟
محیط ایزوله‌شده‌ای که برنامه را با همه وابستگی‌هایش اجرا می‌کند، اما هسته را با سیستم مشترک می‌کند.

## مقایسه VM و Container
| ویژگی | VM | Container |
|-------|-----|-----------|
| ایزوله‌سازی | کامل | سطح فرآیند |
| بوت | دقیقه | ثانیه |
| حجم | گیگابایت | مگابایت |
| سربار | زیاد | کم |

## مفاهیم Docker
| مفهوم | توضیح |
|-------|-------|
| **Image** | قالب فقط‌خواندنی |
| **Container** | نمونه اجرایی Image |
| **Dockerfile** | دستورالعمل ساخت Image |
| **Registry** | مخزن Image‌ها (Docker Hub) |
| **Volume** | ذخیره‌سازی پایدار |
| **Network** | شبکه بین کانتینرها |

## لایه‌های Docker
```
Container (لایه نوشتنی)
    ↓
Image layers (فقط‌خواندنی)
    ↓
Base Image (Ubuntu, Alpine, ...)
```
