# فصل ۲ - ناوبری و فایل‌سیستم | خلاصه

## استاندارد FHS
ساختار پوشه‌های لینوکس طبق استاندارد FHS (Filesystem Hierarchy Standard) تعریف شده:

| پوشه | کاربرد |
|------|---------|
| `/` | ریشه همه چیز |
| `/bin` | دستورات پایه سیستم |
| `/etc` | فایل‌های پیکربندی |
| `/home` | پوشه کاربران |
| `/var` | داده‌های متغیر و لاگ |
| `/tmp` | فایل‌های موقت |
| `/usr` | نرم‌افزارهای نصب‌شده |
| `/proc` | اطلاعات فرآیندها (مجازی) |
| `/sys` | اطلاعات سخت‌افزار (مجازی) |
| `/dev` | فایل‌های دستگاه |
| `/mnt` | نقاط مانت موقت |
| `/opt` | نرم‌افزارهای اختیاری |
| `/root` | پوشه خانگی root |

## مسیر مطلق و نسبی
- **مسیر مطلق**: از ریشه شروع می‌شود → `/home/user/docs`
- **مسیر نسبی**: از پوشه جاری → `../docs` یا `./script.sh`

## نمادهای مسیر
| نماد | معنی |
|------|------|
| `.` | پوشه جاری |
| `..` | پوشه والد |
| `~` | پوشه خانگی کاربر |
| `-` | پوشه قبلی (در cd) |

## Wildcards
| نماد | معنی | مثال |
|------|------|------|
| `*` | هر تعداد کاراکتر | `ls *.txt` |
| `?` | دقیقاً یک کاراکتر | `ls file?.log` |
| `[abc]` | یکی از کاراکترها | `ls file[123].txt` |
| `[a-z]` | محدوده کاراکتر | `ls [a-z]*.conf` |
